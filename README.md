​🎓 مشروع بوصلة الخريجين | Alumni Compass Platform
​من المفهوم إلى المنتج الجاهز (Execution Blueprint)
​هذا المستودع هو المقر الرئيسي لإدارة وتطوير منصة بوصلة الخريجين، وهي منصة متكاملة تهدف لربط الخريجين بالموجهين المهنيين باستخدام تقنيات الذكاء الاصطناعي.  
​👥 فريق العمل والأدوار البرمجية
​تم توزيع الأدوار لضمان التنفيذ المتوازي وتحقيق أقصى كفاءة خلال 6 أسابيع:  
​S1 - قائد هندسة الواجهة الخلفية (Backend Lead): مسؤول عن Laravel API، تصميم قاعدة البيانات، ونظام الصلاحيات.  
​S2 - قائد هندسة الواجهة الأمامية (Frontend Lead): مسؤول عن بناء تطبيق React، الربط مع الـ API، وإدارة الحالة (State).  
​S3 - مهندس الذكاء الاصطناعي والأنظمة (AI Engineer): مسؤول عن محرك التوصيات، فحص السير الذاتية (ATS)، ونظام الاتصال المرئي (WebRTC).  
​S4 - مهندس التكامل وضمان الجودة (QA & Integration): مسؤول عن لوحة تحكم الإدارة، اختبارات التكامل، وعمليات النشر (Deployment).  
​S5 - مهندس UI/UX ودعم Frontend: مسؤول عن نظام التصميم (Shadcn/ui)، الواجهات المتوافقة مع العربية، ودعم تنفيذ الشاشات المعقدة.  
​🚀 خطة الأسبوع الأول: التأسيس والتجهيز (Phase 1)
​الهدف: إزالة أي غموض تقني، تجهيز بيئات العمل، والتأكد من عدم وجود عوائق لدى أي عضو في الفريق.  
​📋 مهام الطلاب (خطوة بخطوة):
​Student 1 (Backend Lead)
​إنشاء منظمة GitHub والمستودعات الثلاثة (Backend, Frontend, AI).  
​تأسيس مشروع Laravel وتجهيز ملفات الهجرة (Migrations) لقاعدة البيانات الأساسية.  
​إعداد ملف .env.example وصياغة "عقد الـ API" الأول لمشاركته مع فريق الـ Frontend.  
​Student 2 (Frontend Lead)
​تأسيس مشروع React باستخدام Vite وتثبيت المكتبات الأساسية (Zustand, Axios, React Router).  
​إعداد هيكل المجلدات وتفعيل نظام اللغة العربية (RTL Layout) باستخدام i18next.  
​بناء عميل Axios (AxiosClient) مع عنوان URL الأساسي للـ API.  
​Student 3 (AI & Systems Engineer)
​تأسيس مشروع FastAPI وربطه بقاعدة بيانات PostgreSQL.  
​إنشاء نقطة نهاية (Endpoint) لاختبار حالة الخدمة /health.  
​البحث في LanguageTool API وتخطيط هيكل بيانات محرك التوصيات.  
​Student 4 (Integration & QA Engineer)
​إعداد GitHub Actions لفحص الكود (Lint) والاختبارات تلقائياً عند كل Pull Request.  
​تجهيز بيئات الاستضافة الأولية (Railway للـ Backend و Vercel للـ Frontend).  
​تجهيز نموذج موحد لتوثيق الـ API لضمان التوافق بين الفريق.  
​Student 5 (UI/UX Engineer)
​إنهاء تصاميم Wireframes لـ 12 شاشة أساسية والحصول على موافقة الفريق.  
​تجهيز نظام التصميم (Tailwind + shadcn/ui) داخل مشروع React.  
​تحديد ملف "Design Tokens" الذي يشمل الألوان، الخطوط، والمسافات الموحدة.  
​🏁 معايير انتهاء المرحلة (Exit Criteria)
​لا يعتبر الأسبوع الأول منتهياً إلا بتحقيق النقاط التالية:  
​[ ] جميع المستودعات الثلاثة جاهزة على GitHub بهيكلية فروع صحيحة.  
​[ ] مشروع Laravel يعمل محلياً مع ربط قاعدة البيانات.  
​[ ] مشروع React يعمل محلياً ويدعم الواجهة العربية (RTL).  
​[ ] مشروع FastAPI يعمل ويستجيب لنقطة النهاية /health.  
​[ ] الموافقة النهائية من الفريق على جميع التصاميم (Wireframes).
